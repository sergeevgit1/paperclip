# Plans

В этой папке хранятся roadmap и планы по развитию проекта.

## Что здесь хранить

- roadmap по развитию продукта и платформы
- технические планы стабилизации
- планы миграций и крупных изменений
- тематические планы по отдельным подсистемам

## Где хранить новые документы

Все новые roadmap и планы нужно сохранять только в `doc/plans/`.

Папку `plans/` в корне репозитория использовать не нужно.

## Формат именования файлов

Новые файлы нужно называть с указанием даты создания и краткого slug-описания.

Шаблон:

```text
YYYY-MM-DD-slug.md
```

Примеры:

```text
2026-04-04-company-knowledge-base-and-file-library.md
2026-04-04-platform-stability-roadmap.md
2026-04-05-agent-provisioning-hardening.md
```

## Исторические файлы

В папке уже могут находиться старые документы с историческими именами, которые не соответствуют текущему шаблону.

Это допустимо для уже существующих файлов.

Правило для дальнейшей работы такое:

- старые файлы не нужно массово переименовывать без отдельной необходимости
- все новые планы и roadmap нужно создавать только в формате `YYYY-MM-DD-slug.md`

## Как с этим работать

- один файл = один цельный план или roadmap
- в имени файла всегда указывать дату создания
- использовать короткий и понятный slug по теме документа
- если план существенно пересматривается, создавать новый файл с новой датой, а не перетирать исторический документ
- в начале файла кратко фиксировать цель документа
- внутри файла держать приоритеты, этапы, критерии завершения и риски
- если документ устарел, это можно явно отметить в самом файле, но не удалять историю без необходимости

## Рекомендуемая структура

- цель
- контекст
- этапы или приоритеты
- конкретные задачи
- риски и зависимости
- критерии завершения
